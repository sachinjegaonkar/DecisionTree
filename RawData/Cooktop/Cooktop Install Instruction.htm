Firewalls

Cooktop runs a small HTTP server on a non-standard port for implementation reasons, such as retrieving stylesheet files.  If you have XP SP2 installed with the firewall on, it will ask if you want to unblock Cooktop.  If you say no, Cooktop probably won't function correctly.  We have not heard of any issues related to the HTTP server but there is always the possibility of buffer overrun exploits if anyone ever thought Cooktop was worth attacking, which is very unlikely.

Please Read this First!

Cooktop 2.5 saves files in the editor, without asking, before running various tools such as XSLT transformations and validation.

Assume that Cooktop will overwrite your file with the current contents of the editor at any given time!

Make a backup of your XML and XSLT files handy before you open them in Cooktop.

We agree that it would be better if Cooktop wrote to temporary files instead, but as many of you know, Cooktop is no longer being developed.  Cooktop is free for a reason (it has only had a single developer), and if you need something better let us suggest the excellent yet clunky (it's a Java app, after all) oXygen that is reasonably priced compared to the others.
